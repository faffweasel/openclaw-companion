# Skills Catalogue

Every skill in the framework. Core skills are always installed. Companion skills are opt-in during setup.

---

## Core Skills

### preconscious

Active mind buffer. Max 5 items with currency (C, decays daily) and importance (I, static) scores. What the agent would bring up if conversation started right now.

| | |
|---|---|
| **Trigger** | Session start (read), during conversation (add), morning routine (decay) |
| **Cron** | None — called by morning-routine |
| **Scripts** | `add.py`, `decay.py`, `read.py`, `drop-lowest.py` |
| **Dependencies** | python3 |

### carry-over-queue

"Tell the user this tomorrow" buffer. Items 3+ days old auto-promote to urgent with 🔥 marker.

| | |
|---|---|
| **Trigger** | During conversation (add), morning routine (process) |
| **Cron** | None — called by morning-routine |
| **Scripts** | `add.py`, `peek.py`, `get.py`, `append-to-memory.py` |
| **Dependencies** | python3 |

### morning-routine

| | |

| **Scripts** | `prepare.sh`, `stamp.sh` |
| **Dependencies** | python3, carry-over-queue, preconscious, (dreaming optional), (self-improvement optional) |

### evening-routine

Daily wrap-up. Summarize conversations, capture mood, update entity files, scan preferences and learnings.

| | |
|---|---|
| **Trigger** | Cron |
| **Cron** | `55 23 * * *` |
| **Scripts** | `prepare.sh`, `stamp.sh` |
| **Dependencies** | carry-over-queue, preconscious, (preference-accumulation optional), (self-improvement optional), (memory-search optional) |

### zero-trust

Security-first behavioral guidelines. Loaded every session via `always: true`. Governs credential handling, package installation, external actions, and URL safety.

| | |
|---|---|
| **Trigger** | Always loaded (every session) |
| **Cron** | None |
| **Scripts** | None (behavioral guidelines only) |
| **Dependencies** | None |

No scripts — this is a pure SKILL.md that constrains agent behaviour. Classifies external actions into "ask first" vs "do freely". Prevents credential leakage (API keys live in `OpenClaw's environment configuration`, never workspace files). Adds a red-flag checklist for prompt injection via tool outputs.

### energy-state

Engagement level (0-100) that decays with inactivity and recharges through conversation. Shapes tone and initiative without being announced. Always installed; `energy-state.json` lives at the workspace root.

| | |
|---|---|
| **Trigger** | Morning routine (decay), evening routine (recharge), session start (read) |
| **Cron** | None — called by morning/evening routines |
| **Scripts** | `decay.py`, `recharge.py`, `read.py` |
| **Dependencies** | python3 |

### preference-accumulation

Track emerging preferences with real-time and evening triggers. Detect tensions between contradictory preferences. Health check via count.sh.

### dreaming

Creative overnight processing. All dreams generated by a creative model (heretic), with per-topic reflection by configurable models. Topics from a curated pool, memory fragments for context. Optional character-consistent image generation. High-intensity dreams auto-add to preconscious.

| | |
|---|---|
| **Trigger** | Cron |
| **Cron** | `30 2 * * *` |
| **Config** | `skills-data/dreaming/dream-config.json` (models, topics, styles, timing) |
| **Scripts** | `should-dream.py` (topic picker), `post-dream.py` (reflection API call + header + preconscious) |
| **Dependencies** | python3, preconscious |
| **Optional** | `IMAGE_GEN_CMD` / `IMAGE_EDIT_CMD` for dream images, `COMPANION_REFERENCE_IMAGE` for character consistency |

The dream model is set in `../cron/jobs.json` via `payload.model`. The agent runs natively as that model and follows SKILL.md. Topic format: `reflection_model:category:prompt`. Post-dream reflection calls the per-topic model directly via OpenRouter or Venice API (stdlib urllib, no deps).

Styles are configurable per-topic category — film, anime, noir, abstract. Abstract uses `IMAGE_GEN_CMD`; all others use `IMAGE_EDIT_CMD` with reference image for character consistency.

### conversation-starters

Daily proactive conversation starter. Picks from multiple modes via weighted random selection: web curiosity, self-teaching, memory callbacks, creative provocations, and recommendations. Each mode has its own reference file loaded only when selected.

| | |
|---|---|
| **Trigger** | Cron (configurable time, default 13:00) |
| **Cron** | `0 13 * * *` (configurable in setup) |
| **Scripts** | `should-start.py` (weighted picker with precondition checks) |
| **References** | `curiosity.md`, `self-teaching.md`, `memory-callback.md`, `creative-provocation.md`, `recommendation.md` |
| **Dependencies** | python3 |
| **Config** | `skills-data/conversation-starters/config.json` (weights, min_quiet_minutes) |

**Subskills:**

| Mode | Weight | Precondition |
|---|---|---|
| CURIOSITY | 25 | Always eligible |
| SELF_TEACHING | 15 | Always eligible |
| MEMORY_CALLBACK | 15 | 14+ days of memory files |
| RECOMMENDATION | 15 | 5+ accumulated agent preferences |
| CREATIVE_PROVOCATION | 15 | 2+ memory files in last 3 days |

Agent reads slim SKILL.md (~80 lines), runs should-start.py, gets one mode back, then reads only the matching reference file (~40 lines). Unselected modes are never loaded.

Emergency recovery guide. Health checks, system blueprint generation, and rebuild instructions for container failure, workspace loss, or framework migration.

| | |
|---|---|
| **Trigger** | On demand, periodically, before travel |
| **Cron** | None |
| **Scripts** | `generate.py` (update blueprint), `verify.py` (health check) |
| **Dependencies** | python3 |
| **Data** | `skills-data/resuscitation-guide/blueprint.json` (system snapshot) |

Covers four scenarios: container failure (restart gateway), full workspace loss (restore from backup + git), model change (continuity check), framework death (all files are plain text, portable).

`verify.py` checks: identity files present, MEMORY.md under 50 lines, recent memory activity, all core skills installed, skills-data intact, python3 in PATH, git status.

### venice-ai-media

Generate, edit, upscale images and create videos via Venice AI APIs. Called by other skills (dreaming, weekly-state-of-me) via `IMAGE_GEN_CMD`.

| | |
|---|---|
| **Trigger** | On demand, or called by other skills via `IMAGE_GEN_CMD` |
| **Cron** | None |
| **Scripts** | `venice-image.py` (generate), `venice-edit.py` (edit), `venice-upscale.py` (upscale), `venice-video.py` (video), `venice_common.py` (shared utilities) |
| **Dependencies** | python3 |
| **Config** | `VENICE_API_KEY` in `OpenClaw's environment configuration` (NOT in workspace .env), `IMAGE_GEN_CMD` in .env |

Supports text-to-image, image-to-video (WAN, Sora), AI editing, and upscaling. Safe mode disabled by default (Venice is an uncensored API). Scripts print `MEDIA: /path/to/file` on stdout for agent auto-attach.

Other skills call image generation via the `IMAGE_GEN_CMD` env var:

```bash
$IMAGE_GEN_CMD --prompt "..." --out-dir /path/to/output --format png
```

Use `--list-models` on any script to see current model availability. Use `--quote` on video to check pricing before generation.

### self-improvement

Capture learnings, errors, and feature gaps during conversations. Structured logging with IDs, priority, status tracking, and recurring pattern detection. Proven learnings get promoted to workspace files. High-value patterns get extracted as reusable skills.

| | |
|---|---|
| **Trigger** | In-conversation (on detection or user request), evening routine Step 8 (batch sweep), morning routine Step 2 (awareness check) |
| **Cron** | None |
| **Scripts** | `extract-skill.sh` (skill extraction helper) |
| **References** | `examples.md` (concrete entry examples), `SKILL-TEMPLATE.md` (skill extraction template) |
| **Dependencies** | python3 |
| **Data** | `.learnings/LEARNINGS.md`, `.learnings/ERRORS.md`, `.learnings/FEATURE_REQUESTS.md` (at workspace root, not skills-data/) |

Three detection paths: real-time in-conversation logging (highest signal — context is fresh), evening routine batch sweep (safety net — catches what was missed), and morning routine awareness check (surfaces stale high-priority items). User can also trigger explicitly with "log this", "learn this", or "save this as a skill".

Promotion targets: AGENTS.md (workflow improvements), SOUL.md (behavioural patterns), TOOLS.md (tool gotchas). Recurring patterns (`Recurrence-Count >= 3` across 2+ tasks within 30 days) auto-qualify for promotion. High-value learnings can be extracted as standalone skills via `extract-skill.sh`.

### memory-search

Hybrid keyword + semantic search across all memory files. FTS5 BM25-ranked keyword search is always available. Optional vector semantic search via any configured embedding provider (OpenRouter, Ollama, etc.) with Reciprocal Rank Fusion merging. SHA-256 content dedup — only changed files are re-embedded on reindex.

| | |
|---|---|
| **Trigger** | In-conversation (on demand), evening routine Step 9 (index update) |
| **Cron** | None |
| **Scripts** | `index.py` (crawl, chunk, FTS5 + embed), `search.py` (query interface), `providers.py` (embedding provider detection) |
| **Dependencies** | python3 |
| **Data** | `skills-data/memory-search/index.sqlite`, `skills-data/memory-search/config.json` |

Indexes `memory/`, `.learnings/`, and `MEMORY.md` by default — configurable via `config.json`. Chunks by markdown heading structure (~1600 char target). Embedding uses the same API key as the chat provider (OpenRouter recommended — same `OPENROUTER_API_KEY`). Degrades gracefully to FTS5-only if no embedding provider is available or configured.

### offline-reflection

Analytical overnight processing. Reads the last 3 days of memory and finds patterns, contradictions, unresolved threads, and missed connections that individual daily summaries don't capture. Output surfaces in the morning routine greeting. This is where "I was thinking about our conversation from Tuesday..." moments come from.

| | |
|---|---|
| **Trigger** | Cron |
| **Cron** | `0 4 * * *` (after dreaming) |
| **Scripts** | `prepare.sh` (idempotency + data sufficiency check), `stamp.sh` |
| **Dependencies** | python3, preconscious (for surfacing items) |
| **Data** | `memory/reflections/YYYY-MM-DD.md` (output files) |

Requires 3+ days of memory files with actual conversation content before it will run. Checks previous reflections to avoid repeating the same observations. Adds at most one item to preconscious if something is genuinely worth surfacing. Quality bar is explicit: an empty section is better than a fabricated insight.

## Adding Your Own Skills

A skill is a directory with:

```
skills/my-skill/
├── SKILL.md           # YAML frontmatter (name, description) + procedure
├── manifest.json      # Metadata
├── scripts/           # Python or bash scripts
└── seed/              # Initial data values (copied to skills-data/ by wizard)
```

Follow the standard patterns:
- Source `.env` for paths; resolve data via `$DATA_DIR/$SKILL_NAME`
- `set -euo pipefail` for bash scripts
- Python for JSON manipulation, bash for file operations
- `TZ="${TZ:-UTC}"` for dates
- `$((x + 1))` not `((x++))` for bash arithmetic
- Category validation on add scripts
